Form types

List form types

Users with administrator rights can list all form types and edit the form type attributes, export and threshold settings and update pattern information.

The form type list can be filtered by entering a form type name in the filter box above the form type name column.

The list information is:

  • Remaining – number of form copies that is still not used and that can be sent to the system
  • Used – number of form copies sent to the system
  • Total – the total number of form copies covered by the pattern licenses for the form type.

Open a form type for editing by clicking the form type row.


Add form type

Users with administrator rights can add new form types to the system. It is very important to set some of the attributes correct from the beginning since there is no way to change those settings afterwards.

Start with entering a form type name following these naming conventions:

The characters a-z, A-Z, 0-9, -, _ and period (.)
The name has to start with a-z,A-Z,-,_ or period.

Browse for the zip-file file that contains the necessary information about the new form type. The file should contain the PAD file, a PDF file and the background images for each page, as PNG.. The zip-file might also contain an xml-file with dynamic data.

After clicking save you have to enter the following settings:
  • Enable print on demand - Can not be changed afterwards!
  • Display period (days) - the number of days completed form copies will be listed in Form copies – Completed.
  • Admin mail notification level (%) – the level of remaining form copies for the form type when the system will notify users with role Admin.
  • Allow paper based verification – If end user verification fields shall be corrected in router or on paper. Only applies for system setups using an interpretation tool.
  • Allow multiple completions – Can not be changed afterwards!
  • Allow multiple pens - Can not be changed afterwards!
  • Export method – Mail or folder export. Shall exported images/data files be sent by email or be stored on the ASH server?
  • From – the from email address when using mail export
  • To – the to email addresses (can be several separated by comma) when using mail export
  • File format – the format for the data export (or none)
  • Image format – the format for the image export (or none)

When pressing save you will get a verification page with a summary of the entered settings. The possibility to break the process still exists.

Thresholds (Only for system setups using an interpretation tool):
For each form type it is possible to define the interpretation threshold that shall be used by the interpretation engine to determine if the interpretation results need user verification or not.
The thresholds can be set for a field or on form type level using predefined levels (set in Settings – Threshold settings).

All set thresholds are presented with the most general setting at the top and the most specific setting at the bottom.

No Field Level
1.AllMedium
2.Card noStrict

A user sends in a form copy:
Name shall be handled as Very tolerant (3)
Card no shall be handled as Strict (2)
All other fields shall be handled as Medium (1)


Form copies

Inbox

In the inbox it is possible to view all form copies received by the system that needs some kind of action.

The inbox can be filtered by entering information in any of the filter boxes above each column.

Open a form copy for viewing details or for being able to take action by clicking the form copy row.

Different states:

    Pidget not ticked
    a form copy is received but the pen user has not ticked the pidget. The form copy will not be processed further until pidget is ticked and the form sent in again, or if a user with admin/verifier role forces the action "Tick pidget"?
    Fields missing
    a form copy is received but the pen user has not filled in all mandatory fields. The form copy will not be processed further until all mandatory fields are filled in and the form sent in again, or if the form copy user or a user with admin/verifier role forces the action "Process anyway"?.
    Verification needed (Only in system setups using an interpretation tool)
    a form copy is received and has passed interpretation. The interpretation result needs verification by the form copy owner or by another user with admin/verifier role. After verification the export process will take place.

A form copy is locked:
when the Form type attribute "Allow multiple completions" is set to No and pidget is ticked.
when Form type attribute "Allow multiple completions" is set to Yes and user area lock_form has strokes.
Different contents depending on user role.
User
a user can only view and take action upon his/her own form copies in state Missing fields or Verification needed.
Verifier
a verifier can view and take action upon all form copies in all states.
Admin
an administrator can view and take action upon all form copies in all states.


Inbox – Pidget not ticked

Users with verifier/admin roles can view form copies in state Pidget not ticked and enforce a "Tick pidget"? activity to let the form copy proceed even if the pen user has not ticked the pidget on the form copy.


Inbox – Fields missing

Users with verifier/admin roles can view form copies owned by all users in state Fields missing and enforce a “Proceed anyway? activity to let the form copy proceed even if mandatory fields are not filled out on the form copy.

Users with User role can only view and take action upon his/her own form copies.


Inbox – Verification needed

Users with verifier/admin roles can view form copies owned by all users in state Verification needed and verify the interpretation results.

Users with User role can only view and verify the interpretation result for his/her own form copies.

Verification is done for one form copy page at the time with possibility to move between the pages via Next and Previous buttons. When moving between the fields that need verification an image zoom for the area on the form copy will be shown to simplify the verification. When all fields are verified, press Send and the final process form the form copy will take place.
If, for some reason, it is needed to view the complete image of the form, just click the page address and all pages of the form copy will be shown in a new tab/window.


Completed

In Form copies - Completed it is possible to view all form copies received by the system that has passed the complete process. The list only shows the form copies sent in during the display period set for each form type.

The list can be filtered by entering information in any of the filter boxes above each column.

Open a form copy for viewing details by clicking the form copy row.

Users with verifier/admin roles can view form copies owned by all users in state Completed and change status to "Pidget not ticked" by pressing the "Mark as not completed" button, if there is a need to open a form copy for more strokes.

Users with User role can only view his/her own form copies. No possibility to change status.

Unlock a user.
Only an administrator can unlock a locked user. When doing so an email with user credentials is sent to the user with a new randomly created password.
Updating Role:
It is possible to change role for all users except the logged in user.
Deleting a user.
It is possible to delete all users except for the logged in user. If a user has form copies belonging to him/her the user will be hidden from the user list. The user name will be modified so that it is possible to still identify the user in form copy lists but at the same time it is clear that the user does no longer exist in the system. The Pen serial will be cleared from the hidden user to make sure it can be used by someone else.

Threshold levels

Users with administrator rights can define threshold levels in system setups using an interpretation tool.

By default the following levels are installed:

Verify all (1.5, 1.5)
Very strict (0.99, 1.0)
Strict (0.86, 0.7)
Medium (0.8, 0.5)
Tolerant (0.66, 0.3)
Very tolerant (0.53, 0.1)
No verification (0, 0)

The threshold levels are used when defining the interpretation thresholds per form type. The interpretation thresholds are used by the interpretation engine to decide if the interpretation result needs to be verified by a user or if the interpretation is considered safe. Each threshold can be set to different levels between 0 and 1. The lower values the more tolerant decisions by the interpretation engine. A threshold level can also be set to a value higher than 1; any value higher than 1 indicates that verification will always be needed, no matter of matching result.
Seven thresholds are predefined. You can change their values and add new thresholds.

NameName of the threshold. 2-20 characters and can only contain the characters a-z, A-Z, 0-9, blank and connector characters as _,#,-
Normalized HWRUsed for handwritten text, reflects how much better the chosen interpretation candidate is than the next one. The higher value set, the more outstanding should the interpretation candidate be from other interpretations.
Resemblance HWRUsed for handwritten text, reflects how well the interpretation matches the digital ink data. The higher value set, the better match is required to consider the interpretation safe.
Resemblance MKRUsed for checkboxes, reflects how well the interpretation matches the digital ink data. The higher value set, the better match is required to consider the interpretation safe.

Support

From the support page it is possible to send an email to the registered support mail address. Enter your email address (if logged in it is shown automatically), a subject and a text describing the problem.

A copy of your support mail will be sent to your own email address as well as to the support mail address.
